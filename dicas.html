Tutoriais, dicas e comandos Git/Github explicados por um Estagiário:


//Lista de comandos utilizados no Git e Windows Terminal//


git init : Cria um repositório Git padrão, tudo o que for feito de alteração dentro desta pasta que agora é um repositorio Git, o mesmo irá monitorar/acompanhar.



git init --bare : É utilizado como repositório remoto local da própria máquina ou servidor, nele não é permitido realizar commits ou executar outras ações. Serve apenas como um local remoto no qual o usuário salva suas alterações, sem precisar enviar para o Github. Você inicializa um repositório como bare quando deseja que ele seja o respositório central.



git remote : Usado para mostrar seus repositórios remotos existentes.



git remote add : Usado para adicionar o seu projeto a um repositório remoto, um local para onde enviar sua modificações seja ele uma Url como por exemplo o GitHub ou um servidor remoto local criado através do git --bare. Sua sintaxe funciona da seguinte forma: git remote add + nome do repositorio a ser criado + o caminho ex. C:/Users/Documents/Git... ou http://github.com/...



git remote -v : Mostra os caminhos dos repositórios existentes.



git clone : Serve para clonar um repositório, para trazer ele pela primeira vez para sua máquina. Sua sintaxe é: git clone + o caminho do repositório a ser clonado.





git push : Utilizado para enviar as modificações do repositório local para o repositório remoto. Sua sintaxe é: git push + o nome do repositório remoto (aquele que vai receber os arquivos) + o branch de onde está enviando.



git pull : Utilizado para trazer as modificações do repositório remoto para o repositório local. Sua sintaxe é: git pull + o nome do repositório remoto a ser recebido + o branch que vai receber.



git branch : Utiliza-se para criar um novo branch(ramo,setor), assim facilitando na organização da construção do projeto. Sua sintaxe é: git branch + o nome da branch a ser criada.



git checkout : Utilizado para navegar/alternar entre as branch. Sua sintaxe é: git checkout + nome da branch que deseja acessar.



gitcheckout -- : Utilizado para desfazer uma alteração que ainda não foi adicionada pelo comando git add ( exemplo da sintaxe: git checkout -- index.html )



git reset : Utilizado para desfazer uma alteração que já foi adicionada pelo comando git add ( exemplo da sintaxe: git reset + o estado (ex. HEAD) + o nome do arquivo que deseja desfazer a alteração (ex. index.html).



git revert : Usado para desfazer um commit. Sua sintaxe é: git revert + o hash do commit (ex. 97323eed32321). Ele desfaz e gera um outro commit informando o que foi desfeito.



git merge : Serve para unirmos as branchs,por exemplo se temos várias branchs num projeto e ao final queremos unificá-lo à branch main utilizamos o git merge + nome da branch a ser mesclada. Utilizando esta forma, todos os commits feitos da branch que estamos recebendo são perdidos e no lugar delas somente mostrará o commit do git merge. Nota-se que devemos estar dentro da branch main para podermos absorver as outras ramificações do projeto.



git rebase : Assim como o merge, serve para unir as informações, porém com o rebase não perdemos os commits das branchs que estamos recebendo, dito isto, o nosso último commit da branch main fica como último commit, e abaixo dele nos é mostrado os commits da branch que puxamos.



git log --graph : Mostra um gráfico de todos os commits feitos em todas as branchs, torna o entendimento mais fácil podemos associar até a uma famosa frase: "Entendeu ou quer que eu desenhe?" ^^.



git status : Mostra o status do repositório, como por exemplo se foi alterado algum arquivo, se falta commitar algo etc.



git log : Mostra todos os logs de alterações feitas naquele repositório.



git log --oneline : Mostra todos os logs de alterações feitas naquele repositório linha a linha de forma resumida.



git log -p : Mostra todos os logs de alterações feitas naquele repositório com todos os detalhes, um log mais completo.



git add : Serve para adicionar as alterações feitas num arquivo específico. Muito útil quando se trabalha com várias branchs em paralelo.



git add . : Serve para adicionar as alterações feitas em todos os arquivos de forma geral. Exemplo, se você fizer alterações no cabeçalho, no rodapé porém trabalha em um único branch, você pode usar este comando e rapidamente salvar todas alterações, sem ter que ficar digitando, nome de arquivo por arquivo que foi modificado ou criado.



git commit -m : Serve para descrever um alteração feita no projeto/arquivo ou seja toda alteração feita usa-se o "commit -m" para deixar descrito o que foi alterado, para facilitar o trabalho em equipe ou mesmo quando for dar continuidade no projeto se achar mais rápido onde parou tipo um checkpoint.



git config --local user.name "nome do usuário" : Utilizado para definir o usuário que está enviando as alterações no projeto em questão.



git config --global user.name "nome do usuário" : Utilizado para definir o usuário que está enviando as alterações em qualquer projeto realizado no computador em questão.



git config --local user.email "nome do usuário" : Utilizado para definir o email do usuario que está enviando as alterações no projeto em questão.



git config --global user.email "nome do usuário" : Utilizado para definir o email do usuário que está enviando as alterações em qualquer projeto realizado no computador em questão. Para visualizar o nome de usuario e de email que está logado no gitbash, basta utilizar o comando git config user.name e/ou git config user.email



mkdir + "nome da pasta": Usado para criar pastas.



cd : Usado para navegar entre as pastas.



q : Utilizado para sair do git log -p.



:x : Utilizado para sair da tela do git merge ou rebase



git tag -a : Utilizado para criar uma release, uma versão inalterável do projeto. Sua sintaxe é: git tag -a + o nome que deseja dar a sua versão do programa e se quiser adicionar uma mensagem de descrição para deixar mais claro a versao que está sendo disponibilizada é só colocar "-m" ou seja git tag -a v1.0.1(sugestao de nome para a tag) -m "mensagem de descrição".



git blame : É usado para saber o responsavel por determinado commit, sua sintaxe é: git blame + o nome do arquivo no qual deseja saber os responsaveis pelas alterações.



git show : Mostra todas as alterações aplicadas pelo commit com o hash informado. Sua sintaxe é: git show + o hash do commit.



//Entendendo os comandos do GitHub//


Issue : Utilizado para reportar problemas no código, sugerir melhorias e organizar quaisquer coisas que façam sentido para o projeto.



Pull Requests : Serve para solicitar o envio de alterações no código no qual deseja realizar alterações melhorias e/ou correções.



Forks : Serve para copiar um repositório na qual esta cópia vira um clone do estado atual do repositório, fazendo assim com que você possa experimentar mudanças e features(recursos) novas sem precisar utilizar o repositório principal e poder realizar as melhorias.



//Existem vários serviços que disponibilizam a hospedagem de sites de forma gratuita aqui vou disponibilizar uma lista de sites com estes serviços//


GitHub Pages

O GitHub Pages é um serviço do GitHub, que permite a hospedagem de sites utilizando HTML|CSS|JavaScript gratuitamente. Basta enviar os arquivos para o repositório e seguir as instruções dadas pelo GitHub. Embora o GitHub Pages seja uma excelente opção para sites estáticos, ele não dá suporte a linguagens de back-end, como PHP, Ruby, Python, C# entre outras.

GitLab Pages

Assim como o GitHub, o GitLab também oferece um serviço para hospedagem de sites estáticos, que utilizam somente HTML/CSS/JS.

BitBucket Pages

O BitBucket também oferece uma hospedagem gratuita para sites estáticos, o BitBucket Pages. Para publicar o site no BitBucket Pages, é necessário criar um repositório e enviar os arquivos HTML/CSS/JS.

Surge

Surge disponibiliza a hospedagem de sites estáticos com domínio personalizado e certificado SSL de forma gratuita. Para quem quer hospedagem um site estático simples, a plataforma da Surge é uma excelente opção.
Embora a utilização do Surge é por meio de um programa com interface em linha de comando, a utilização é muito simples, basta digitar um único comando para publicar o site na Web.

Vercel

Vercel é uma plataforma para frameworks front-end e sites estáticos, onde se pode hospedar gratuitamente seus sites, também possui a forma paga. Na Vercel se tem o desempenho como padrão, além de suportar Angular, React, Vue.js, Sveltekit, Next.js entre outros. Possui integração com o Github.

Netlify

Com a hospedagem de sites estáticos da Netlify é possível hospedar projetos pessoais ou comerciais, com CDN e certificado SSL.
A hospedagem gratuita da Netlify também oferece recursos para trabalhar com formulários, que geralmente é uma limitação nos sites estáticos. Além da rápida implementação, é capaz de fazer o deploy com apenas alguns cliques. Possui integração com o Github.

000webhost

000webhost é bastante conhecida por oferecer serviços de hospedagem de sites há bastante tempo. Além de poder hospedar um site estático na 000webhost, também é possível hospedar sites dinâmicos que utilizam PHP, MySQL, cPanel e sem anuncios. Possui integração com o Github.
